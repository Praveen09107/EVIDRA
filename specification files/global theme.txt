Global Visual Theme (applies to all pages)
Overall style: Dark forensic cockpit, with glass cards, subtle neon accents and strong focus+context visualizations (graph + timeline) as recommended for complex analytical tools.

Layout pattern:

Left fixed sidebar for main navigation.

Top bar for case context, search, and pipeline controls.

Main canvas center, contextual inspector on the right.

Interactions:

Consistent navigation gestures: click to select, scroll to zoom, drag to pan timelines, standard keyboard shortcuts (arrows to move timelines, +/- to zoom) to keep interactions intuitive and familiar.

Hover tooltips on all charts/graphs to avoid label clutter and keep readability.

Page 1 — Command Center (Global Multi‑Agent Dashboard)
Layout
Left sidebar: Global nav (Command Center, Cases, Agents, Labs, Replay, Reports, Analytics, Settings).

Main grid: Three vertical zones:

Top: System Overview Strip (full width)

Row of small, pill‑style cards:

“Total Cases Today”, “Agents Running”, “Avg Pipeline Latency”, “Failures Last 24h”.

Each card shows a large number and a mini sparkline.

Middle left: Agent Status Grid

3×4 grid of Agent cards (one per agent).

Each card:

Icon + agent name.

Status dot (green/orange/red).

Small metrics row: avg latency, calls today, error rate.

“Open Agent Lab” button.

Card background uses subtle elevation and color to signal status (e.g., red halo for failing agents).

Middle right: Live Activity Stream

Scrollable list:

Row example: “TOD Agent ran on Case #23 in 1.8s — result: 19:45–21:05 window”.

Colored tag for agent name, clickable to open Agent Lab or filter stream by agent.

Date/time stamp aligned to the right.

Bottom: Pipeline Heatmap

Matrix: rows = agents, columns = active cases.

Cell color intensity indicates number of runs or current load for that agent on that case.

Clicking a cell opens Case Workspace focused on that agent’s contribution to that case.

Key Interactions
Hover on agent card → show tooltip with last 3 runs (cases + outcomes).

Clicking an agent name → go to that agent’s dedicated Agent Lab (Page 7).

Clicking a case name in Activity Stream or Heatmap → go to Case Workspace (Page 3).

Filter toggle in the stream (chips): “Errors only”, “Slow runs”, “Anomaly-heavy cases”.

Visual Feel
Very “ops dashboard”: dark background, strong contrast metrics, minimal but expressive color palette, aligning with multi‑agent ops dashboards recommended in practice.

Page 3 — Case Workspace (Master Investigation Cockpit)
This is your hero screen.

Overall Layout
Top band: Case header.

Below header: Multi‑Agent Pipeline Strip.

Main canvas (center): three tabs (Timeline, Hotspots, Causal Graph).

Right inspector panel: contextual details.

3.1 Case Header
Left: Case title, victim info, location.

Center: Hypothesis chips:

Horizontal list of pills: “Natural”, “Accident”, “Suicide”, “Homicide”.

Each pill: probability bar inside (e.g., 72% fill), colored by risk.

Right:

“Run full pipeline” / “Re‑run changed agents” buttons.

Pipeline run status indicator (e.g. “Last run: 3 minutes ago”).

3.2 Multi-Agent Pipeline Strip
Horizontal stepper (like a DAG simplified in a single row):

Nodes: Autopsy → TOD → Timeline & Anomaly → Correlation → Claim Extractor → Evidence–Claim Mapper → Hypothesis Manager → Bias Monitor → Replay Engine.

Each node:

Circular icon with agent initials and status ring (animated border when running).

Below: small label “Done” / “Running” / “Warning”.

Edges:

Subtle animated arrows showing flow direction.

Clicking a node:

Highlights that agent’s contributions in the current tab.

Opens a small popover:

Last run summary, link to full Agent Lab.

3.3 Tab 1 — Timeline View
Layout (top→bottom):

Main timeline (top third of center)

Large chart showing event volume over time with colored stacked bars/lines by source category (CCTV, phone, IoT, autopsy events).

TOD window visualized as a translucent vertical band across the plot.

Anomaly intensity overlay (e.g., heat gradient behind the curve in periods where AE score is high).

Mini timeline (immediately under main)

Slim overview timeline spanning entire case period.

Draggable/resizeable window rectangle controlling the zoomed region in the main timeline (classic focus+context pattern).

Event lanes (middle third)

Horizontal lanes stacked:

Autopsy/TOD.

Phone metadata.

CCTV events.

Other sensors/logs.

Agent outputs (e.g., anomaly markers, collisions).

Each event: small pill or icon anchored at its timestamp, color-coded by type. Labels only appear when zoomed in enough to keep readability.

Event table (bottom third)

Synchronized list view of events in the current time window.

Columns: Time, Source, Event type, Short description, Anomaly score, Link to raw data.

Interactions:

Scroll wheel / pinch on main timeline → zoom in/out; mini timeline updates focus window accordingly.

Drag window in mini timeline to pan time range.

Hover over events:

Shows tooltip with details and highlights the corresponding row in the table.

Clicking an event:

Selects it, opens details in the right inspector panel.

Keyboard shortcuts:

Arrow keys to move time window, +/- to zoom, ensuring accessible navigation.

3.5 Tab 3 — Causal / Hypothesis Graph
Layout:

Central canvas: node-link diagram:

Hypothesis nodes (larger, top).

Claim nodes (middle).

Evidence nodes (bottom).

Right: explanation panel.

Interactions (based on graph UX best practices):

Focus mode: dims everything else, highlights its supporting and contradicting paths.

Inspector shows:

Current probability.

List of strongest supporting / contradicting evidences with links back to timeline.

Zoom & pan:

Standard mouse wheel zoom and drag to pan.

Zoom indicator suggests when to zoom further when labels get dense.

Hover:

Short tooltips on nodes/edges (no overcrowding).

Toggle:

Buttons for “Show only critical evidence”, “Show all evidence”.

3.6 Right Inspector Panel
Dynamic content:

For selected event: metadata, source, associated agent outputs.

For hotspot: involved agents, time windows, detailed explanation, “open in XAI studio” button.

For graph node: explanation why evidence supports/contradicts claim, with confidence and bias notes.

Page 4 — Autopsy & TOD Lab
Layout
Two-column split:

Left: Autopsy Intelligence.

Right: TOD Engine
Components:

PMI distribution chart

A smooth density plot over time (x-axis = time, y-axis = probability).

Overlaid markers:

“Physics only” estimate.

“Hybrid” estimate.

TOD 95% confidence band shaded.

Feature importance bar chart

Horizontal bars: ambient temp, clothing, BMI, environment, etc.

Bars sorted by importance; labels on left, % on right.

Mode switcher

Segment control: “Physics”, “Hybrid”, “Signs-constrained”.

Switching fades between charts, updating text explanation.

Consistency widget

Cards: Rigor, Livor, Algor vs estimated TOD.

Interactions:

Clicking a “Signs-constrained” mode explains exactly which biological signs tightened the window.

Interactions:

5.2 Anomaly Explorer
Table with each row representing a time window (e.g. 15 minutes):

Window start–end.

Autoencoder reconstruction error (AE score).

IF score.

Fused anomaly score.

Dominant features contributing to anomaly (e.g., “long silence”, “night-time unusual movement”).

Side mini-chart:

Distribution histogram of all anomaly scores (so high values stand visually apart).

Interactions:

Click a row:

Highlights corresponding window in Digital Timeline (e.g., with a translucent band).

Hover over a row:

Shows a small popover with quick narrative (e.g., “20-min phone silence during TOD band”).

Filter controls:

Slider for minimum anomaly score.

Checkbox “Intersect TOD window”.

5.3 CCTV / Scene Viewer
Primary video panel:

Video frame with YOLO/DETR boxes.

Color-coded boxes by entity, ID labels (“Person #3”, “Vehicle #2”).

Timeline scrub bar:

Below the video, segments colored for:

High anomaly.

Collision hotspot times.

Small pins marking detections or suspicious intervals.

Entity list (side panel):

Each row: thumbnail + label + presence segments (mini timeline bars showing when they appear on which camera).

Interactions:

Hover on entity in list → highlight only that entity’s boxes in video.

Clicking a hotspot segment in scrub bar → jump to that time, link back to Hotspots tab in Case Workspace.
