AIVENTRA Frontend Spec v1.0
0. Global Design System
0.1 Viewport & Breakpoints
Primary target: Desktop (1366–1920 px width), 768+ px height.

Breakpoints:

lg (≥ 1280 px): Full 3‑column layout (sidebar + main + inspector).

md (1024–1279 px): Sidebar collapsible; inspector collapsible.

sm (≤ 1023 px): Single-column stacked views with tabs; inspector becomes modal.

All specs below assume lg breakpoint; for md/sm, the agent should adapt by stacking main and inspector or collapsing the sidebar into an icon bar.

0.2 Layout Structure
Left navigation sidebar:

Width: 80 px (collapsed), 240 px (expanded; optional).

Fixed, full height.

Main content column:

Width: calc(100vw - sidebar_width - inspector_width).

Padding: 24 px left/right, 20 px top/bottom.

Right inspector panel:

Width: 320 px.

Fixed, full height minus global header.

Scrollable.

0.3 Color Palette (Dark Forensic Theme)
Backgrounds:

bg-base: #05060A (page background).

bg-elevated: #0F1015 (cards & surfaces).

bg-glass: rgba(255, 255, 255, 0.04) (glass panels).

Borders & dividers:

border-subtle: rgba(255, 255, 255, 0.06).

Text:

text-primary: #F9FAFB.

text-muted: #9CA3AF.

text-faint: #6B7280.

Accents & statuses:

Primary accent (analysis): #22C7D5 (cyan).

Secondary accent (selection/hover): #38BDF8.

Anomaly/high-risk: #F97373 (warm magenta/red).

Warning: #FBBF24.

Success: #22C55E.

Semantic:

TOD band: rgba(34, 199, 213, 0.18).

Anomaly heat: rgba(251, 113, 133, 0.22).

Background detail:

Apply subtle noise texture overlay on bg-base at 3–5% opacity.

Apply soft vignette (darkening at edges) to keep attention toward center.

0.4 Typography
Display font (titles): Instrument Serif or Playfair Display.

H1 (page title): 28–32 px, weight 600–700.

Body font (UI): Inter or Plus Jakarta Sans.

Body text: 14–16 px, 1.5 line height.

Meta text: 12 px.

Rules:

Titles and major labels always left-aligned, avoid centered text for reading.

Use at most:

4 font sizes: 32 (H1), 20 (H2), 16 (body), 12 (meta).

Colors:

Primary text: text-primary.

Secondary text / meta: text-muted / text-faint.

0.5 Spacing & Radius
Spacing scale (px): 4, 8, 12, 16, 24, 32.

Cards: padding 16–24 px.

Section gaps: 24–32 px.

Border radius:

Small pill: 9999 px.

Cards: 12–16 px.

Node icons: circle (50%).

Completed: cyan.

Labels:

Interactions:

Click:

Emphasize contributions:

When pipeline is executing:

Node rings animate clockwise; edges show a subtle shimmering “flow pulse” from left to right.

3.5 Timeline Tab — Detailed Layout
Vertical distribution (within main content area)
Main timeline: 30%.

Mini timeline: 10%.

Event lanes: 30%.

Event table: 30%.

3.5.1 Main Timeline (TimelineChart component)
Full width minus 8 px margin to inspector edge.

Height: ~220–260 px.

Visual elements:

Background: bg-elevated with slight inner shadow.

Plot area:

Stacked area or stacked bars; colors:

CCTV = blue.

Phone = purple.

Other = gray.

Anomaly heat:

Semi-transparent red/pink band behind the curve wherever fused anomaly score > threshold.

TOD band:

Vertical translucent cyan rectangle.

Axes:

X-axis:

Ticks, but minimal labels (e.g. hour ticks).

Y-axis:

No explicit labels by default; only shown on hover or when zoomed deep (to reduce clutter).

Interaction details:

Hover anywhere on chart:

Vertical line (cursor) drawn at that x.

Tooltip at top:

Time (exact).

Total events.

Breakdown by source.

Average anomaly score.

“Inside TOD window” badge if within band.

Zoom:

Scroll/pinch horizontally changes x-axis range.

Smooth transitions; mini timeline updates focus.

Double-click:

Reset zoom to entire case or current mini-window.

3.5.2 Mini Timeline (MiniTimeline component)
Height: 40 px.

Same width as main timeline.

Contents:

Compressed density line.

TOD band again.

Focus window:

Rectangular overlay with draggable edges.

Interaction:

Drag the entire window:

Pan the visible portion.

Drag left/right edges:

Zoom in/out.

Click outside window:

Recenter window around click.

3.5.3 Event Lanes (EventLane component)
Number of lanes: 4–5.

Layout:

Each lane height: 40–60 px.

Stacked vertically, with 8 px gap.

Lane composition:

Left label area: 80 px.

Label text (“CCTV”, “Phone”, “Autopsy/TOD”, etc.).

Small icon in lane color.

Right plot area:

Horizontal strip from left label edge to right boundary.

Each event is placed according to timestamp.

Events:

Visual shape:

Discrete event: small pill (width 10–16 px) with optional micro-icon if zoomed in enough.

Interval event: bar spanning a time range.

Color:

Base color by source.

Outline/halo for anomalies.

Interactions:

Hover event:

Show tooltip detailing event.

Highlight row in Event table.

Click event:

Mark as selected (solid outline).

Inspector shows full details (metadata, raw content if safe, related claims).

Drag to select:

User can click-drag in lane to define time window; this becomes a filter across lanes and table.

Interactions:
